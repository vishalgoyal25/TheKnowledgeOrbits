## SYSTEM OVERVIEW

### How the System Works

```
┌─────────────────────────────────────────────────────────────────┐
│                    CONTENT INGESTION PIPELINE                    │
└─────────────────────────────────────────────────────────────────┘

STEP 1: STATIC CONTENT (NCERT Books)
───────────────────────────────────
PDF Upload → Text Extraction → Chunking (1200 chars)
→ Embedding Generation (384-dim) → Store in Database

Tables Created:
- content_document (1 record per PDF)
- content_chunk (~200-300 chunks per book)
- content_embedding (1 per chunk)

┌─────────────────────────────────────────────────────────────────┐

STEP 2: KNOWLEDGE STRUCTURE (Syllabus)
──────────────────────────────────────
Manual/Admin Creates:
- Programs (UPSC CSE)
- Subjects (Polity, History, Geography...)
- Modules (Fundamental Rights, Indian Constitution...)
- Topics (Right to Equality, Article 14...)

Tables Created:
- knowledge_program
- knowledge_subject
- knowledge_module
- knowledge_topic

┌─────────────────────────────────────────────────────────────────┐

STEP 3: CHUNK-TOPIC MAPPING (Linking)
─────────────────────────────────────
Admin Maps Chunks → Topics
- Select chunk
- Assign to relevant topic(s)
- Store mapping

Tables Created:
- knowledge_chunk_topic_map (many-to-many)

Now: Topics have linked chunks!

┌─────────────────────────────────────────────────────────────────┐

STEP 4: CURRENT AFFAIRS INGESTION
─────────────────────────────────
RSS Scraping → Article Extraction → Chunking (1200 chars)
→ Embedding Generation → Store in Database

Tables Created:
- ca_source (The Hindu, Indian Express)
- ca_article (raw news articles)
- ca_chunk (processed chunks)
- content_embedding (1 per CA chunk)

┌─────────────────────────────────────────────────────────────────┐

STEP 5: AUTO CA-TOPIC LINKING
─────────────────────────────
Semantic Similarity Algorithm:
- Get CA chunk embedding (384-dim)
- Calculate topic embeddings (average of top 10 mapped chunks)
- Compute cosine similarity
- Link if similarity > 0.65 (threshold)

Tables Created:
- ca_topic_link (CA chunk → Topic mapping)

Now: Topics have both static chunks + CA chunks!

┌─────────────────────────────────────────────────────────────────┐

STEP 6: ARTICLE GENERATION
──────────────────────────
For a given topic:
1. Fetch static chunks (from chunk_topic_map)
2. Fetch CA chunks (from ca_topic_link, last 30 days)
3. Assemble context:
   - Section 1: THEORETICAL FOUNDATION (static chunks)
   - Section 2: CURRENT CONTEXT (CA chunks)
4. Send to GROQ API (LLM)
5. Generate narrative article
6. Store article + source attribution

Tables Created:
- article_article (generated articles)
- article_source_map (chunk → article mapping)

Result: Integrated article with theory + current affairs!
```

```python
# More sophisticated: Use chapter names + keywords
python manage.py shell

from engines.knowledge.models import Topic, ChunkTopicMap
from engines.content.models import Chunk

# Define topic → keyword/chapter mapping
topic_mappings = {
    "Right to Equality": {
        "keywords": ["equality", "article 14", "article 15", "discrimination"],
        "chapters": ["Fundamental Rights", "Equality"]
    },
    "Right to Freedom": {
        "keywords": ["freedom", "article 19", "article 20", "article 21", "article 22"],
        "chapters": ["Fundamental Rights", "Freedom"]
    },
    # ... more topics
}

for topic_name, criteria in topic_mappings.items():
    topic = Topic.objects.get(name__icontains=topic_name.split("(")[0].strip())

    # Find chunks matching keywords OR chapters
    chunks = Chunk.objects.filter(
        chunk_text__icontains=criteria["keywords"][0]
    ) | Chunk.objects.filter(
        chapter_name__icontains=criteria["chapters"][0]
    )

    chunks = chunks[:15]  # Limit per topic

    for chunk in chunks:
        ChunkTopicMap.objects.get_or_create(
            chunk=chunk,
            topic=topic,
            defaults={'relevance_score': 1.0}
        )

    print(f"Mapped {chunks.count()} chunks to '{topic.name}'")

exit()
```

## CONTENT LINKING PROCESS

### Understanding the Linking Algorithm

**Static Content Linking (Manual):**

```
Admin → Selects Chunk → Assigns to Topic → Creates ChunkTopicMap
Reason: Human judgment ensures accuracy for foundational content
```

**CA Content Linking (Automatic):**

```
1. Get CA chunk embedding (384-dim vector)
2. For each topic:
   a. Calculate topic embedding = average of top 10 mapped chunks
   b. Compute cosine similarity(CA chunk, topic embedding)
3. If similarity > 0.65:
   a. Create CATopicLink
   b. Store relevance_score = similarity
4. Max 3 topics per chunk (highest scores)
```

**Why It Works:**

- Topics already have static chunks mapped (Phase 3)
- Topic embeddings represent the "semantic center" of the topic
- CA chunks similar to topic center → likely relevant
- Threshold 0.65 balances precision vs recall

**Example:**

```
Topic: "Right to Equality"
Mapped Static Chunks: 15 chunks from NCERT about equality, Article 14-18
Topic Embedding: Average of 15 chunk embeddings

CA Chunk: "Supreme Court today upheld reservation for economically
weaker sections under Article 15(6), stating it doesn't violate
equality principles..."

Similarity Calculation:
cosine_similarity(CA chunk embedding, Topic embedding) = 0.78

Result: Link created! (0.78 > 0.65 threshold)
```

## ARTICLE GENERATION

### Understanding Article Generation

**How It Works:**

```
Input: Topic ID + include_ca flag

Step 1: Fetch Static Chunks
- Query: ChunkTopicMap.filter(topic_id=topic_id)
- Limit: Top 10 chunks (by relevance_score)
- Source: NCERT textbooks

Step 2: Fetch CA Chunks (if include_ca=True)
- Query: CATopicLink.filter(topic_id=topic_id, published_at > 30 days ago)
- Limit: Top 5 chunks (by relevance_score)
- Source: Recent news articles

Step 3: Assemble Context
Context = """
=== THEORETICAL FOUNDATION (NCERT) ===

[Chunk 1 text]
Source: NCERT Polity, Page 45

[Chunk 2 text]
Source: NCERT Polity, Page 47

... (up to 10 chunks)

=== CURRENT CONTEXT (Recent Developments) ===

[CA Chunk 1 text]
Source: The Hindu, January 2026

[CA Chunk 2 text]
Source: Indian Express, February 2026

... (up to 5 chunks)
"""

Step 4: Generate with GROQ
Prompt = """
You are an expert UPSC educator.
Generate a comprehensive article on: {topic_name}

Source Material:
{context}

Requirements:
- Introduction (100 words)
- Body with theory + current examples (600 words)
- Conclusion with exam perspective (100 words)
- Total: 800-1000 words
"""

Response = GROQ API call (llama-3.1-70b-versatile)

Step 5: Store Article
- Create Article record
- Create ArticleSourceMap records (links to source chunks)
- Mark generation_type = 'ai_generated'
- Calculate read_time, word_count

Result: Integrated article saved to database!
```

# Find chunks without embeddings
chunks_without_embeddings = Chunk.objects.exclude(
    id__in=Embedding.objects.filter(content_type='chunk').values_list('content_id', flat=True)
)

print(f"Chunks missing embeddings: {chunks_without_embeddings.count()}")

for chunk in chunks_without_embeddings:

    Embedding.objects.create(
